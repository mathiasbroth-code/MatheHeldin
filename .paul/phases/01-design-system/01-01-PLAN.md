<objective>
## Goal
Design System "Clean Minimal" (Richtung 3) als wiederverwendbare UI-Primitives implementieren. Alle gemeinsamen Komponenten (Button, Card, Input, ProgressBar, StufeKarte, FeedbackBanner, TippButton) und das App-Layout nach den verbindlichen Design-Specs aus SPECIAL-FLOWS.md.

## Purpose
Jede Stufe, jede Seite der App verwendet dieselben UI-Primitives. Ohne ein sauberes Design System werden Farben, Radii, Fonts und Tap-Targets in jeder Stufe neu erfunden. Das Design System ist die Grundlage für Phase 2–7.

## Output
- Tailwind-Theme-Konfiguration mit den Clean-Minimal-Farben
- 7 wiederverwendbare UI-Komponenten in src/components/ui/
- App-Layout-Shell mit Header und Navigation
- Uebersicht.tsx refactored auf die neuen Komponenten
</objective>

```gherkin
Given die Button-Komponente mit variant="primary" oder variant="secondary"
```

## AC-3: Card-Komponente rendert korrekt
```gherkin
Given die Card-Komponente
When sie gerendert wird
Then hat sie bg #f8fafc, 1px solid #e2e8f0 Border, rounded-2xl, KEINEN box-shadow
```

## AC-4: Input-Komponente mit Focus-Ring
```gherkin
Given die Input-Komponente
When sie Focus erhält
Then zeigt sie Teal-Border + 3px Ring
And Zahlen rendern mit tabular-nums
```

## AC-5: ProgressBar zeigt Fortschritt
```gherkin
Given die ProgressBar-Komponente mit value=3 und max=5
When sie gerendert wird
Then zeigt sie einen 8px hohen Balken, Track #e2e8f0, Fill #14b8a6 bei 60%
```

## AC-6: StufeKarte mit Kategorie-Farbe
```gherkin
Given eine StufeKarte mit icon, titel, sub, farbe und fortschritt
When sie gerendert wird
Then zeigt sie das Icon in einem farbigen Kreis, Titel bold, Sub grau, Mini-ProgressBar
And die ganze Karte ist als Touch-Target klickbar (min 44px Höhe)
```

## AC-7: FeedbackBanner zeigt richtig/falsch
```gherkin
Given FeedbackBanner mit typ="richtig"
When gerendert
Then grüner Hintergrund #f0fdf4 mit Erfolgsmeldung
Given FeedbackBanner mit typ="falsch"
When gerendert
Then warmer Hintergrund #fffbeb mit "🤔 Noch nicht ganz" + Hinweistext
```

## AC-8: Layout-Shell mit Header
```gherkin
Given die AppShell-Komponente
When eine Seite darin gerendert wird
Then zeigt sie einen Header mit "Mathe-Heldin" Titel und optionalen Zurück-Button
And der Content-Bereich hat max-width und ist zentriert
And safe-area-padding für Notch-Geräte
```

## AC-9: Build fehlerfrei
```gherkin
Given alle neuen Komponenten
When pnpm build ausgeführt wird
Then kompiliert TypeScript ohne Fehler
And Vite baut erfolgreich
```
