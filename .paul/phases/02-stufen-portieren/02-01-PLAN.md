<objective>
## Goal
Wire up the StufeView page to render actual stage exercises, and port the first stage (`mitte` — "Zahl in der Mitte") as end-to-end proof that the complete pipeline works: generator → View → answer check → Dexie persistence → progress display.

## Purpose
The `mitte` stage is the simplest standalone stage with the richest interactions (Zahlenstrahl, Tipp, Input). If this works end-to-end, all other stages can follow the same pattern. This plan also makes the app actually *usable* for the first time.

## Output
- `src/stages/mitte/` — complete stage module
- `src/pages/StufeView.tsx` — wired to render any registered stage
- `src/stages/registry.ts` — with `mitte` registered
- App shows one stage card on the overview, clicking it opens a working exercise

## Phase 2 Overview (this is Plan 1 of 4)
- **02-01**: StufeView wiring + `mitte` stage (THIS PLAN)
- **02-02**: `lesen` stages (mengen20, bis100lesen, bis1000lesen, bis10000lesen) — shared ZahlLesenView
- **02-03**: `legen` + `buendeln` + `zerlegen` — shared ZahlLegenView + 2 standalone
- **02-04**: `skizze` — complex 3-phase flow
</objective>
