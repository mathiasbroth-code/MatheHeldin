<objective>
## Goal
Add bottom tab navigation (Üben | Fortschritt | Eltern) and create the Fortschritt and Eltern-PIN pages. After this, the app has full navigation for kids and a gated entry point for parents.

## Purpose
Tab navigation makes the app feel like a real mobile app. The Fortschritt page gives children immediate pride in their progress. The Eltern-PIN gate prepares the foundation for the Phase 6 dashboard.

## Output
- Bottom TabBar component (3 tabs)
- Fortschritt page with basic per-stage stats
- ElternGate page with PIN entry (placeholder dashboard behind it)
- AppShell updated to accommodate TabBar
</objective>

## AC-1: TabBar shows on main screens
```gherkin
Given a profile is active
When the user is on /ueben or /fortschritt
Then a bottom TabBar with 3 tabs is visible: Üben, Fortschritt, Eltern
And the active tab is highlighted (primary color)
And each tab has an icon + label
And tap targets are ≥ 44px
When the user is on /stufe/:id or /profil/neu or /
Then the TabBar is NOT shown
```

## AC-2: Fortschritt page shows per-stage progress
```gherkin
Given the user navigates to /fortschritt
Then a header shows "[Avatar] Dein Fortschritt"
And a summary card shows total stages completed (richtig ≥ zielRichtige)
And a list of all stages shows: icon, titel, progress bar (richtig/zielRichtige), percentage
And stages with 100% show a checkmark
And stages with 0% show "Noch nicht geübt"
```

## AC-3: ElternGate requires PIN
```gherkin
Given the user taps "Eltern" tab
When no PIN is set yet
Then a PIN-setup screen asks to create a 4-digit PIN
When PIN is set
Then a PIN-entry screen appears with 4 digit inputs
When correct PIN is entered
Then a placeholder dashboard shows "Eltern-Dashboard kommt in Phase 6"
When wrong PIN is entered
Then feedback says "Falscher PIN" and inputs clear
```

## AC-4: PIN persisted in Zustand
```gherkin
Given a PIN is set
When the app is reloaded
Then the PIN is still remembered (persist middleware)
And the Eltern tab still requires PIN entry
```

## AC-5: AppShell accommodates TabBar
```gherkin
Given TabBar is shown
Then main content has bottom padding so it's not hidden behind the fixed TabBar
And TabBar is fixed to the bottom with safe-area padding
```
