<objective>
## Goal
Build the aufgaben infrastructure: TypeScript types matching the YAML schema, a parser that loads Markdown+YAML into typed objects, and a pool system that serves random aufgaben filtered by stage/difficulty.

## Purpose
This decouples exercise content from stage code. Authors write Markdown, the parser converts it, the pool serves it to stages. 500 existing aufgaben become instantly usable.

## Output
- `src/aufgaben/types.ts` — full type system for all 8 interaction types
- `src/aufgaben/parser.ts` — YAML+Markdown → typed AufgabenPool
- `src/aufgaben/pool.ts` — query interface (by stage, difficulty, type)
- `src/aufgaben/index.ts` — barrel export

- **04-01**: Types + Parser + Pool (THIS PLAN)
- **04-02**: 8 Interaktionstyp-View-Komponenten

</objective>

## AC-1: Complete type system
```gherkin
Given the 8 interaction types from FORMAT.md
Then TypeScript interfaces exist for each:
  - EingabeAufgabe (numeric input)
  - AuswahlAufgabe (multiple choice with options[])
  - ZuordnungAufgabe (match pairs[][])
  - LueckeAufgabe (template string with gaps[])
  - ReihenfolgeAufgabe (items[] to sort)
  - SchrittAufgabe (steps[] with sequential inputs)
  - WahrFalschAufgabe (statement + correct boolean)
  - TextaufgabeAufgabe (context text + sub-questions)
All extend a common BankAufgabe base with: titel, typ, thema, schwierigkeit, buchseite, kapitel, stage_id, tipps[3], loesung, loesungsweg, didaktischerHinweis
```

## AC-2: Parser converts Markdown to typed objects
```gherkin
Given a Markdown file like docs/aufgaben/01-wiederholung.md
When parseAufgabenDatei(content) is called
Then it returns an array of typed BankAufgabe objects
And YAML frontmatter is parsed into metadata fields
And Markdown sections (Aufgabenstellung, Lösung, Lösungsweg, Tipp 1/2/3) are extracted
```

## AC-3: Pool provides filtered random access
```gherkin
Given parsed aufgaben are loaded into the pool
When pool.getRandom({ stageId: 'bis2000lesen', schwierigkeit: 'gelb' }) is called
Then it returns a random aufgabe matching the filters
When pool.getAll({ kapitel: '01-wiederholung' }) is called
Then it returns all aufgaben from that chapter
When no aufgaben match
Then it returns null (getRandom) or empty array (getAll)
```

    ```
    Schwierigkeit = 'gelb' | 'grün' | 'orange'
    InteraktionsTyp = 'eingabe' | 'auswahl' | 'zuordnung' | 'luecke' | 'reihenfolge' | 'schritt' | 'wahr-falsch' | 'textaufgabe'
    
    BankAufgabeBase {
      titel: string
      typ: InteraktionsTyp
      thema: string
      schwierigkeit: Schwierigkeit
      buchseite: number
      kapitel: string
      stageId: string
      digital: 'voll' | 'teilweise' | 'platzhalter'
      aufgabenstellung: string
      loesung: string
      loesungsweg: string
      tipps: [string, string, string]
      didaktischerHinweis?: string
    }
    
    // Per-type extensions with structured data parsed from aufgabenstellung/loesung:
    EingabeAufgabe extends BankAufgabeBase { typ: 'eingabe', items: { frage: string, antwort: string }[] }
    AuswahlAufgabe extends BankAufgabeBase { typ: 'auswahl', optionen: string[], richtigeOption: number }
    ZuordnungAufgabe extends BankAufgabeBase { typ: 'zuordnung', paare: { links: string, rechts: string }[] }
    LueckeAufgabe extends BankAufgabeBase { typ: 'luecke', template: string, luecken: string[] }
    ReihenfolgeAufgabe extends BankAufgabeBase { typ: 'reihenfolge', items: string[], richtigeReihenfolge: number[] }
    SchrittAufgabe extends BankAufgabeBase { typ: 'schritt', schritte: { anweisung: string, antwort: string }[] }
    WahrFalschAufgabe extends BankAufgabeBase { typ: 'wahr-falsch', aussage: string, richtig: boolean, korrektur?: string }
    TextaufgabeAufgabe extends BankAufgabeBase { typ: 'textaufgabe', kontext: string, fragen: { frage: string, antwort: string }[] }
    
    BankAufgabe = EingabeAufgabe | AuswahlAufgabe | ... (discriminated union on typ)
    ```

    NOTE: The structured per-type fields (items, optionen, paare, etc.) are parsed from the Markdown content.
    For v1, we store the raw aufgabenstellung/loesung strings AND attempt structured parsing.
    If structured parsing fails, the View components fall back to rendering raw Markdown.

    **src/aufgaben/parser.ts:**
    - `parseAufgabenDatei(markdown: string): BankAufgabeBase[]`
    - Split on `## Aufgabe N` boundaries
    - For each block:
      - Extract YAML between `---` markers → parse manually (no yaml lib, simple key: value)
      - Extract `### Aufgabenstellung` → aufgabenstellung
      - Extract `### Lösung` → loesung
      - Extract `### Lösungsweg` → loesungsweg
      - Extract `### Tipp 1/2/3` → tipps array
      - Extract `### Didaktischer Hinweis` → didaktischerHinweis
    - YAML parser: simple line-by-line `key: "value"` or `key: value` (no nested objects, no arrays)
      - Handle quoted values: strip surrounding quotes
      - Convert buchseite to number
      - Map stage_id to stageId (snake_case → camelCase)
    - Return BankAufgabeBase[] (structured per-type parsing in a later step)
    
    **src/aufgaben/pool.ts:**
    - `class AufgabenPool`
    - `load(aufgaben: BankAufgabeBase[])` — adds to internal array
    - `getRandom(filter?: { stageId?: string, schwierigkeit?: Schwierigkeit, typ?: InteraktionsTyp, kapitel?: string }): BankAufgabeBase | null`
    - `getAll(filter?: ...): BankAufgabeBase[]`
    - `getCount(filter?: ...): number`
    - Uses randInt for random selection
    - Singleton export: `export const aufgabenPool = new AufgabenPool()`

    **src/aufgaben/index.ts:**
    - Barrel exports
    - `loadAllAufgaben()` async function that:
      - Uses Vite's `import.meta.glob` to load all `docs/aufgaben/[0-9]*.md` files as raw strings
      - Parses each with parseAufgabenDatei
      - Loads into aufgabenPool
      - Returns total count
