<objective>
## Goal
BaumDiagramm-Komponente interaktiv machen: Kind tippt auf "Nächste Ebene"-Button,
um Äste schrittweise aufzudecken, statt alles auf einmal zu sehen.
Nach richtiger Antwort wird der Baum komplett aufgedeckt.

## Purpose
Philippa soll den Baum aktiv erkunden und verstehen, WARUM es N Möglichkeiten gibt.
Schrittweises Aufdecken spiegelt den didaktischen Prozess: erst 1. Platz wählen,
dann sehen wie sich die Optionen für 2. Platz verzweigen, usw.

## Output
- BaumDiagramm mit Tap-to-Reveal (Button "Nächste Ebene aufdecken")
- Zähler zeigt sichtbare Permutationen ("3 von 6 sichtbar")
- Nach richtiger Antwort: automatisch komplett aufgedeckt
</objective>

## AC-1: Schrittweises Aufdecken
```gherkin
Given eine Kombinatorik-Aufgabe mit 3 Elementen wird angezeigt
When das Kind den "Nächste Ebene"-Button tippt
Then wird die nächste Baumebene aufgedeckt (revealLevel erhöht sich um 1)
And der Button verschwindet wenn alle Ebenen sichtbar sind
```

## AC-2: Permutations-Zähler
```gherkin
Given der Baum ist teilweise aufgedeckt
When eine neue Ebene aufgedeckt wird
Then zeigt ein Zähler "X von Y Möglichkeiten sichtbar"
And bei voller Aufdeckung zeigt er "Alle Y Möglichkeiten"
```

## AC-3: Auto-Reveal bei richtiger Antwort
```gherkin
Given das Kind hat die richtige Anzahl eingegeben
When die Antwort als richtig bewertet wird
Then wird der Baum automatisch komplett aufgedeckt
And der Zähler zeigt "Alle Y Möglichkeiten"
```
