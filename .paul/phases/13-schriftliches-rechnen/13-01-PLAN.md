## AC-1: Addition/Subtraktion stellengerecht
```gherkin
Given eine Aufgabe mit stageId "schriftlich-addieren" oder "schriftlich-subtrahieren" oder "schriftlich-addsub-groß"
When die Aufgabe angezeigt wird
Then erscheint über dem Eingabefeld eine vertikale Aufstellung:
  - Zahlen rechtsbündig untereinander, Ziffer für Ziffer in Spalten
  - Operationszeichen (+/−) links neben der unteren Zahl
  - Horizontale Linie unter den Operanden
  - Bei 3 Summanden: alle drei untereinander
```

## AC-2: Multiplikation stellengerecht
```gherkin
Given eine Aufgabe mit stageId "schriftlich-multiplizieren-*"
When die Aufgabe angezeigt wird
Then erscheint eine vertikale Multiplikations-Aufstellung:
  - Faktor 1 oben, Faktor 2 darunter mit · links
  - Horizontale Linie
  - Bei mehrstelligem Multiplikator: Teilprodukte untereinander
```

## AC-3: Division als Rahmenformat
```gherkin
Given eine Aufgabe mit stageId "schriftlich-dividieren-*"
When die Aufgabe angezeigt wird
Then erscheint das deutsche Divisions-Schema:
  - Dividend : Divisor = ___
  - Schrittweise Zerlegung darunter (Teilung, Subtraktion, Rest runter)
```

## AC-4: Kaskaden-Integration
```gherkin
Given eine Aufgabe OHNE schriftlich-Kontext (z.B. Routen, Einheiten)
When die Aufgabe angezeigt wird
Then wird keine SchriftlicheRechnung gerendert
And bestehende Visualisierungen funktionieren unverändert
```

<task type="auto">
  <name>Task 1: SchriftlicheRechnung-Komponente erstellen</name>
  <files>src/aufgaben/views/SchriftlicheRechnung.tsx</files>
  <action>
    Erstelle Parser + Render-Komponente nach dem etablierten Muster (wie StellenwertTafel, RoutenDiagramm):

    **Parser `parseSchriftlicheRechnung(stageId, aufgabenstellung, teilFrage?)`:**
    - Erkennung via stageId: nur bei stageIds die "schriftlich-" enthalten
    - Extrahiert aktuelle Teilaufgabe (z.B. "463 + 215") — der Parser bekommt die Einzel-Frage, nicht die ganze Aufgabenstellung
    - Parsed Operanden und Operator (+, −, ·, :)
    - Entfernt Tausenderpunkte für die Stellenberechnung, zeigt sie aber an
    - Return: `{ operanden: string[], operator: string, typ: 'addition'|'subtraktion'|'multiplikation'|'division' }` oder null

    **Render `<SchriftlicheRechnung>`:**
    
    A) Addition/Subtraktion:
    - Monospace-Grid, jede Ziffer in eigener Spalte
    - Rechtsbündig ausgerichtet (kürzere Zahlen links mit Leerzeichen auffüllen)
    - Operator (+/−) in eigener Spalte links neben der letzten Zahl
    - Horizontale Linie (`border-bottom`) unter den Operanden
    - Ergebniszeile: nur Fragezeichen "?" in gleicher Monospace-Breite
    - Bei 3+ Operanden: alle untereinander, Operator nur bei der letzten
    
    B) Multiplikation:
    - Faktor 1 oben, · + Faktor 2 darunter
    - Horizontale Linie darunter
    - Ergebniszeile mit "?"
    
    C) Division:
    - Deutsches Format: Dividend : Divisor = ?
    - Alternativ: klassisches Rahmenformat mit Dividend links, Divisor rechts vom Strich
    - Einfacher Ansatz: Nur die Aufstellung zeigen, NICHT die Zwischenschritte (die macht das Kind)

    **Styling:**
    - `font-variant-numeric: tabular-nums` + Monospace für Ziffern-Ausrichtung
    - Gleiche Card-Ästhetik wie StellenwertTafel
    - Kompakte Darstellung, nicht zu groß (Aufgabe ist die Hauptsache)
    - Tausenderpunkte in der Anzeige beibehalten
  </action>
  <verify>
    - TypeScript kompiliert fehlerfrei: `pnpm tsc --noEmit`
    - Export: `parseSchriftlicheRechnung` und `SchriftlicheRechnung` sind benannt exportiert
  </verify>

<task type="auto">
  <name>Task 2: AufgabeWrapper-Kaskade erweitern</name>
  <files>src/aufgaben/views/AufgabeWrapper.tsx</files>
  <action>
    SchriftlicheRechnung in die Kaskade einbauen:

    1. Import: `import { SchriftlicheRechnung, parseSchriftlicheRechnung } from './SchriftlicheRechnung';`
    
    2. Parser-Aufruf NACH stellenwertDaten, VOR kreiseDaten (logische Position):
       - Problem: Die Aufgabenstellung enthält alle Teilaufgaben (a/b/c), aber die Visualisierung soll die aktuelle zeigen
       - Lösung: Parser bekommt aufgabenstellung + stageId. Wenn stageId matcht, returned er `{ stageId }` als Marker
       - Die Komponente selbst entscheidet dann anhand der aktuellen Teilaufgabe was angezeigt wird
       
       ALTERNATIV (einfacher, bevorzugt):
       - Parser liefert die erste Rechnung als Default-Anzeige
       - Die aktuelle Teilaufgabe wird via EingabeView/SchrittView dynamisch angezeigt
       - Nachteil: Viz zeigt immer a) auch wenn Kind bei c) ist
       
       EMPFEHLUNG: Die SchriftlicheRechnung-Komponente bekommt die volle Aufgabenstellung UND die aktuelle Teilfrage (falls verfügbar via Props-Drilling oder Context). Start mit einfachem Ansatz: Zeige nur die aktuelle Rechenaufstellung.

    3. Kaskaden-Guard: `!routenDaten && !einheitenDaten && !divisionsDaten && !werteBalkenDaten && !stellenwertDaten && !kreiseDaten`
    
    4. Rendering: `{schriftlichDaten && <SchriftlicheRechnung {...schriftlichDaten} />}`
    
    WICHTIG: Die Teilaufgaben-Synchronisierung (welche Teilaufgabe gerade aktiv ist) ist eine bekannte Herausforderung. Für Plan 01 reicht ein statischer Ansatz — die Aufstellung zeigt die Struktur der Rechnung, das Kind löst im Eingabefeld. Die dynamische Synchronisierung kann in Plan 02 kommen.
