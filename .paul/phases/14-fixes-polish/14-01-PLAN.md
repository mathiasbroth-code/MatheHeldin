## Bug-Analyse

**Rechenketten:** 2 Aufgaben betroffen. Lösung enthält `−` (U+2212 Unicode Minus),
Regex `[+\-·:×÷]` matcht nur ASCII Hyphen (U+002D). Kette bricht nach 1. Schritt ab.
- "72 → · 5 → 360 → − 20 → 340" → nur 1 Schritt statt 2
- "40 → · 8 → 320 → − 15 → 305" → nur 1 Schritt statt 2

**Teilaufgaben-Sync:** AufgabeWrapper ist stateless, kennt currentLabel nicht.
bankStage.tsx hat den State + handleTeilaufgabeChange. EingabeView/SchrittView
rufen onTeilaufgabeChange auf. Aber AufgabeWrapper leitet nur durch, nutzt es
nicht für parseSchriftlicheRechnung. Parser akzeptiert bereits `teilFrage?`.
</context>

    3. Bei neuer Aufgabe zurücksetzen: `useEffect(() => setActiveLabel(''), [aufgabe.titel])`
    4. Wrapper-Handler erstellen der BEIDES tut:
       ```

         setActiveLabel(label);
         onTeilaufgabeChange?.(label);
       }, [onTeilaufgabeChange]);
       ```
    5. An View-Komponente `handleTeilaufgabeChange` statt `onTeilaufgabeChange` übergeben
    6. Bei parseSchriftlicheRechnung: die aktive Teilfrage ermitteln und übergeben:
       - Aus aufgabe.parsed die items/teilaufgaben holen
       - Das Item mit label === activeLabel finden
       - Dessen `frage` als teilFrage an den Parser geben
       Pseudocode:
       ```
       const aktiveTeilFrage = activeLabel
         ? findAktiveTeilFrage(aufgabe, activeLabel)
         : undefined;
       const schriftlichDaten = ... parseSchriftlicheRechnung(
         aufgabe.stageId, aufgabe.aufgabenstellung, aktiveTeilFrage
       );
       ```

    **Hilfsfunktion `findAktiveTeilFrage`** (in AufgabeWrapper oder inline):
    - Prüfe aufgabe.parsed.items (EingabeDaten) oder aufgabe.parsed.teilaufgaben (SchrittDaten)
    - Finde das Item/die Teilaufgabe mit passendem Label
    - Return die `frage` als String (z.B. "328 + 147")

    **SchriftlicheRechnung.tsx:** Keine Änderungen nötig — der Parser akzeptiert
    bereits `teilFrage?` und bevorzugt es gegenüber aufgabenstellung.
