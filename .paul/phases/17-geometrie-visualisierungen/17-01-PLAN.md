<objective>
## Goal
Isometrische Würfel-Visualisierung für Schrägbilder und Rauminhalt-Aufgaben — das sind die Stages wo Kinder 3D-Würfelgebäude "sehen" müssen, aktuell aber nur Text lesen.

## Purpose
13 Aufgaben (7 schraegbilder + 7 rauminhalt) beschreiben 3D-Würfelgebäude als Text ("Ein L-förmiges Gebäude: 3 Würfel in einer Reihe, 2 obendrauf"). Philippa als visueller Lerntyp braucht ein Bild, keinen Text.

Parkettierungen und Körpernetze sind Multiple-Choice ohne 3D — dort reicht der Text zunächst.

## Output
- `IsometricGrid.tsx`: Wiederverwendbare isometrische Würfel-Darstellung
- AufgabeWrapper-Integration via Kaskade
</objective>

<context>
## Betroffene Stages
- schraegbilder: 7 Aufgaben (typ eingabe) — "Zähle die Würfel im Gebäude"
- rauminhalt: 7 Aufgaben (typ eingabe) — "Wie viele Würfel? Welcher Rauminhalt in cm³?"

## Aufgaben-Muster
Beschreibung in Text: "Ein L-förmiges Gebäude: 3 Würfel in einer Reihe, 2 Würfel obendrauf am linken Ende"
→ Das Kind soll die Würfel zählen

## Visualisierungs-Ansatz
Isometrische Projektion (30°-Winkel) mit gestapelten Würfeln:
- Jeder Würfel als Parallelogramm mit 3 sichtbaren Flächen
- Farben: Oberseite hell, links mittel, rechts dunkel (3D-Effekt)
- Grid-basiert: Position (x, y, z) → SVG-Koordinaten

## Kaskaden-Pattern
Wie bestehende Visualisierungen: Parser erkennt stageId, extrahiert Würfel-Positionen aus dem Aufgabentext, Komponente rendert SVG.
</context>

  <action>
    Erstelle eine wiederverwendbare isometrische Würfel-Visualisierung:

    **Props:**
    ```
    interface IsometricGridProps {
      wuerfel: { x: number; y: number; z: number }[];
      size?: number; // Würfelkantenlänge in px, default 30
    }
    ```

    **Rendering (SVG):**
    - Isometrische Projektion: x-Achse 30° nach rechts-unten, y-Achse 30° nach links-unten, z-Achse vertikal
    - Jeder Würfel hat 3 sichtbare Flächen (top, left, right)
    - Farben: primary-Palette (teal-Töne: top #5eead4, left #14b8a6, right #0d9488)
    - Kanten: 1px stroke in dunklerer Farbe
    - Sorting: Würfel von hinten nach vorne rendern (painter's algorithm)
    - ViewBox auto-berechnen aus Würfel-Positionen

    **Hilfsfunktionen:**
    - `toIso(x, y, z, size)` → {screenX, screenY} — 3D→2D Projektion
    - `cubePolygons(x, y, z, size)` → {top, left, right} Polygon-Punkte
    - `sortCubes(cubes)` → Sortierung für korrekte Überlappung

    Kein externer SVG-Library-Import. Reines React + SVG.
  </action>

    **Parser `parseWuerfelGebaeude(stageId, aufgabenstellung)`:**
    - Nur bei stageId "schraegbilder" oder "rauminhalt"
    - Extrahiert Würfelpositionen aus Text-Beschreibungen
    - Heuristik-basiert: Suche nach Mustern wie:
      - "X Würfel in einer Reihe" → [{x:0,y:0,z:0}, {x:1,y:0,z:0}, ...]

      - "L-förmig", "Turm", "Treppe" → vordefinierte Formen
      - Fallback bei nicht-parsebarem Text: einfacher 2×2×1 Block
    - Return: `{ wuerfel: {x,y,z}[] }` oder null

    **Kaskade in AufgabeWrapper:**
    - Import IsometricGrid + Parser
    - Position: nach schriftlichDaten, vor kreiseDaten
    - Guard: wie üblich mit !routenDaten && !einheitenDaten && ...
    - Render: `{wuerfelDaten && <Card className="py-2 px-3"><IsometricGrid {...wuerfelDaten} /></Card>}`
