<objective>
## Goal
SchriftlicheRechnung von statischer Anzeige zu interaktiver Ziffern-Eingabe upgraden. Das Kind tippt jede Ziffer einzeln in die richtige Spalte — wie im Heft.

## Purpose
60 schriftliche Rechenaufgaben zeigen aktuell nur die Aufstellung mit "?" als Platzhalter. Das Kind muss die Antwort als Ganzes ins Standard-Eingabefeld tippen. Interaktive Spalten-Eingabe ist pädagogisch wertvoller: stellengerechtes Arbeiten wird geübt.

## Output
- SchriftlicheRechnung.tsx: Digit-Input-Boxes in der Ergebniszeile
- Übertrag-Zeile über der Ergebniszeile
- Auto-Submit wenn alle Ziffern gefüllt
</objective>

<context>
## Aktueller Stand
- SchriftlicheRechnung.tsx: Statische Anzeige (Operanden + Linie + "???")
- AufgabeWrapper rendert SchriftlicheRechnung VOR der EingabeView/SchrittView
- Ergebnis wird über das Standard-Eingabefeld eingegeben

## Architektur-Entscheidung
Die interaktive SchriftlicheRechnung ERSETZT die Standard-Eingabe für diese Aufgaben.
AufgabeWrapper erkennt schriftlich-Aufgaben und übergibt onRichtig/onFalsch an
SchriftlicheRechnung statt an EingabeView. Die View unten wird ausgeblendet.

## Betroffene Aufgabentypen
- Addition/Subtraktion: Ergebnis hat maxLen+1 Ziffern (Übertrag)
- Multiplikation: Ergebnis hat maxLen+multiplierLen Ziffern
- Division: Ergebnis-Ziffern = Dividend-Ziffern / Divisor (einfache Ganzzahl-Division)
</context>

<acceptance_criteria>

## AC-1: Ziffern-Eingabe pro Spalte
```gherkin
Given eine schriftliche Addition 463 + 215
When die Aufgabe angezeigt wird
Then erscheinen unter der Linie leere Input-Boxes (eine pro Ergebnis-Ziffer)
And jede Box akzeptiert genau eine Ziffer (0-9)
And nach Eingabe springt der Fokus automatisch zur nächsten Box
```

## AC-2: Übertrag-Anzeige
```gherkin
Given eine Addition mit Übertrag (z.B. 572 + 389)
When das Kind die korrekte Ziffer eingibt
Then erscheint ein kleiner Übertrag-Indikator über der nächsten Spalte
```

## AC-3: Auto-Submit bei korrekter Eingabe
```gherkin
Given alle Ergebnis-Ziffern sind eingetippt
When die letzte Ziffer korrekt ist
Then wird onRichtig() aufgerufen
And die Standard-EingabeView wird NICHT angezeigt
```

## AC-4: Fehler-Feedback
```gherkin
Given das Kind hat alle Ziffern eingetippt
When das Ergebnis falsch ist
Then erscheint "Noch nicht ganz"-Feedback
And die Boxen werden geleert zum erneuten Versuch
```

<task type="auto">
  <name>Task 1: SchriftlicheRechnung interaktiv machen</name>
  <files>src/aufgaben/views/SchriftlicheRechnung.tsx</files>
  <action>
    SchriftlicheRechnung erweitern:

    **Neue Props:**
    ```
    interface SchriftlicheDaten {
      operanden: string[];
      operator: '+' | '−' | '·' | ':';
      typ: 'addition' | 'subtraktion' | 'multiplikation' | 'division';
    }

    interface SchriftlicheRechnungProps extends SchriftlicheDaten {
      interaktiv?: boolean;
      erwartetesErgebnis?: string;
      onRichtig?: () => void;
      onFalsch?: () => void;
    }
    ```

    **Ergebnis berechnen:** Aus Operanden + Operator das Ergebnis intern berechnen.
    Keine Abhängigkeit von der Aufgabe-Lösung — die Rechnung ist eindeutig.

    **Interaktive Ergebniszeile (VertikaleAufstellung):**
    - Statt "?" → Input-Boxes (je 1 Ziffer)
    - Ergebnis-Länge = maxLen + 1 (für möglichen Übertrag bei Add/Sub)
    - Führende Leerzeichen wenn Ergebnis kürzer als maxLen+1
    - Jede Box: `<input inputMode="numeric" maxLength={1} className="w-7 h-8 text-center ...">`
    - Auto-Focus: Nach Eingabe einer Ziffer → nächste Box fokussieren
    - Eingaberichtung: Rechts nach Links (Einer zuerst, wie im Heft)
    - Refs-Array für Focus-Management

    **Übertrag-Zeile:**
    - Zeile über den Operanden mit kleinen Carry-Indikatoren
    - Berechnung: Für Addition stelle für stelle prüfen ob Summe >= 10
    - Anzeige: Kleine "1" über der nächsten Spalte wenn Übertrag entsteht
    - NUR anzeigen wenn die entsprechende Ergebnis-Ziffer bereits eingetippt ist
    - Styling: text-[10px], text-primary/50

    **Submit-Logik:**
    - Wenn alle nicht-leeren Boxen gefüllt → prüfen
    - Vergleich: eingegebene Ziffern vs berechnetes Ergebnis
    - Richtig → onRichtig(), Boxen grün
    - Falsch → onFalsch(), Feedback, nach 1.5s Boxen leeren

    **Division:** Einfacher als Add/Sub — nur horizontales Ergebnis-Input:
    - Dividend : Divisor = [__] (einzelne Input-Box, nicht pro Ziffer)
    - Kein Übertrag nötig
