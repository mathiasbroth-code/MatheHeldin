<objective>
## Goal
Adaptives Lernsystem einbauen: Kompetenzwert pro Stage, Rostfaktor für nicht geübte Themen, automatischer Schwierigkeitsfilter, "Vorgeschlagen für dich" auf der Übersicht, Eltern-Regler (frei/sanft/gezielt).

## Purpose
Philippa bekommt personalisierte Lernempfehlungen basierend auf ihrem tatsächlichen Können. Schwache Themen werden häufiger vorgeschlagen (nicht erzwungen), starke Themen seltener. Eltern steuern die Intensität. Kein ANKI-Klon — themenbasiertes Wiederholen mit neuen Varianten statt identischer Aufgaben.

## Output
- `useAdaptiv()` Hook: berechnet Kompetenz + Rostfaktor pro Stage aus bestehenden AntwortLog-Daten
- `lernmodusStore`: Eltern-Einstellung (frei/sanft/gezielt) per Profil
- "Vorgeschlagen für dich"-Bereich auf der Übersicht
- Automatischer Schwierigkeitsfilter-Default in bankStage
- Eltern-Dashboard-Erweiterung mit Lernmodus-Regler
</objective>

<acceptance_criteria>

## AC-1: Kompetenzwert pro Stage
```gherkin
Given ein Profil mit AntwortLog-Einträgen für verschiedene Stages
When useAdaptiv() aufgerufen wird
Then erhält man pro Stage einen Kompetenzwert (0–1) basierend auf:
  - Gewichteter Durchschnitt der letzten 10 Antworten (richtig ohne Tipp=1.0, mit Tipp=0.5, falsch=0.0)
  - Neuere Antworten wiegen stärker (exponentieller Decay)
  - Rostfaktor: max 0.02 pro Tag seit letzter Übung, gedeckelt bei 0.3
```

## AC-2: Vorgeschlagen-für-dich auf Übersicht
```gherkin
Given Lernmodus ist "sanft" oder "gezielt"
When die Übersicht geladen wird
Then erscheint oben ein "Vorgeschlagen für dich"-Bereich mit 2–3 Stage-Karten
And die Empfehlungen priorisieren: niedrige Kompetenz + hoher Rostfaktor
And Stages die nie geübt wurden erscheinen NICHT (nur bekannte schwache Themen)
```

## AC-3: Automatischer Schwierigkeitsfilter
```gherkin
Given ein Kind öffnet eine Bank-Stage
When Lernmodus "sanft" oder "gezielt" aktiv ist
Then wird der Schwierigkeitsfilter automatisch voreingestellt:
  - Kompetenz < 0.4 → gelb (einfach)
  - Kompetenz 0.4–0.7 → kein Filter (alle)
  - Kompetenz > 0.7 → grün oder orange
And das Kind kann den Filter jederzeit manuell ändern
```

## AC-4: Eltern-Lernmodus
```gherkin
Given ein Elternteil ist im ElternGate eingeloggt
When der Bereich "Lernmodus" angezeigt wird
Then kann zwischen drei Modi gewählt werden:
  - "Frei" — keine Empfehlungen, kein Auto-Filter (Default)
  - "Sanft gelenkt" — Empfehlungen + Auto-Filter, Kind wählt frei
  - "Gezielt" — Empfehlungen prominent, schwache Themen betont
And die Einstellung wird pro Profil gespeichert
```

## AC-5: Kein Zwang, kein Lock
```gherkin
Given beliebiger Lernmodus
When das Kind die Übersicht nutzt
Then sind ALLE Stages immer zugänglich
And Empfehlungen sind Vorschläge, nie Sperren
And der manuelle Schwierigkeitsfilter überschreibt immer den Auto-Filter
```

  <name>Task 1: useAdaptiv Hook + lernmodusStore</name>
  <files>src/hooks/useAdaptiv.ts, src/stores/lernmodusStore.ts, src/db/repository.ts</files>
  <action>
    **lernmodusStore.ts:**
    Zustand-Store mit persist. Pro Profil ein Lernmodus: 'frei' | 'sanft' | 'gezielt'. Default: 'frei'.
    Interface: `getLernmodus(profileId) → Modus`, `setLernmodus(profileId, modus)`.

    **useAdaptiv.ts:**
    Hook der getStufeRepetitionMap() aus repository.ts nutzt (existiert bereits!) und daraus berechnet:

    1. `kompetenz(stage)`: gewichteter Score 0–1
       - Letzte 10 Antworten, neuere wiegen mehr (Gewicht = 0.9^index)
       - richtig + tippStufe=0 → 1.0 | richtig + tippStufe>0 → 0.5 | falsch → 0.0
       - Normalisiert durch Summe der Gewichte

    2. `rostfaktor(stage)`: 0–0.3
       - `min(0.3, tagesSeitLetzterUebung × 0.02)`

    3. `effektiveKompetenz = max(0, kompetenz - rostfaktor)`

    4. `empfohleneStages`: sortiert nach effektiveKompetenz aufsteigend, nur Stages mit totalAntworten ≥ 3 (nie geübte ausfiltern), max. 3 zurück

    5. `schwierigkeitDefault(stageId)`: Schwierigkeit | null
       - effKompetenz < 0.4 → 'gelb'
       - effKompetenz > 0.7 → 'grün'
       - sonst → null (kein Filter)

    Hook gibt zurück: `{ empfohleneStages, schwierigkeitDefault, lernmodus }`
    Wenn lernmodus === 'frei' → empfohleneStages=[], schwierigkeitDefault=immer null.

    **repository.ts:**
    getStufeRepetitionMap() existiert bereits — KEINE Änderung nötig, nur importieren.
    Erweitere getLetzteAntworten() um tippStufe mitzugeben (tut es bereits via AntwortLog).
  </action>
  <verify>npx tsc --noEmit — keine Fehler. Import von useAdaptiv in einer Test-Komponente funktioniert.</verify>
  <done>AC-1 satisfied: Kompetenzwert + Rostfaktor berechnet. AC-4 teilweise: Store existiert.</done>
</task>
