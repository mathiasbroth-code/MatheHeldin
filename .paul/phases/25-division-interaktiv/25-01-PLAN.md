## AC-1: Schriftliche Division — Kästchenpapier interaktiv
```gherkin
Given eine schriftliche Divisions-Aufgabe (z.B. 3632 : 4)
When die Aufgabe angezeigt wird
Then erscheint ein Kästchenpapier-Raster mit:
  - Dividend oben (vorausgefüllt), Divisor rechts, Ergebnis-Kästchen leer
  - T/H/Z/E Stellenwert-Header
And das Kind tippt Schritt für Schritt:
  - Ergebnis-Ziffer für aktuelle Stelle (z.B. 9 für T)
  - System zeigt Produkt (9×4=36) und Minus-Zeichen
  - Kind tippt Rest der Subtraktion (z.B. 0)
  - System holt nächste Ziffer runter
  - Nächste Stelle wird aktiv
And bei Null im Ergebnis (z.B. 3:4=0) muss 0 explizit eingegeben werden
And bei Rest am Ende wird "R X" neben dem Ergebnis notiert
```

## AC-2: Halbschriftliche Division — Freie Zerlegungs-Eingabe
```gherkin
Given eine halbschriftliche Divisions-Aufgabe (z.B. 376 : 8)
When die Aufgabe angezeigt wird
Then erscheint ein Eingabebereich:
  - Oben: "376 : 8 = ___" (Gesamtergebnis)
  - Zeile 1: Kind tippt Teilzahl (z.B. 320) + Teilergebnis (z.B. 40)
  - System prüft: 320 ist Vielfaches von 8, ≤ 376
  - System zeigt Rest: 376 - 320 = 56
  - Zeile 2: "56 : 8 = ___", Kind tippt 7
  - Gesamtergebnis: 40 + 7 = 47, wird oben eingetragen
And verschiedene korrekte Zerlegungen werden akzeptiert
And Validierung prüft: Teilzahl muss Vielfaches des Divisors sein
```

## AC-2b: Halbschriftliche Division — Vielfache-Auswahl (Multiple Choice)
```gherkin
Given eine halbschriftliche Aufgabe MIT Vielfache-Auswahl (z.B. "448:8" mit [80,160,240,320,400,480,560,640,720,800])
When die Aufgabe angezeigt wird
Then erscheint oben die Aufgabe "448 : 8 = ___"
And darunter: "Welches Vielfache wählst du?" mit Auswahl-Buttons
And Kind tippt auf ein Vielfaches (z.B. 320)
And System prüft: 320 ≤ 448, 320 ist Vielfaches von 8
And Danach: freie Eingabe für Rest-Division wie in AC-2
```

## AC-2c: Schriftliche Division — Erste-Zahl-Auswahl (Multiple Choice)

    Neue Komponente `SchriftlicheDivision` erstellen.

    **Algorithmus `berechneDivisionSchritte(dividend: number, divisor: number)`:**
    Berechnet alle Schritte voraus und gibt ein Array zurück:
    ```ts
    interface DivSchritt {
      stelle: number;           // 0=T, 1=H, 2=Z, 3=E
      aktuellerWert: number;    // Die Zahl die dividiert wird (z.B. 36, 03, 32)
      ergebnisZiffer: number;   // Ergebnis für diese Stelle (z.B. 9, 0, 8)
      produkt: number;          // ergebnisZiffer × divisor (z.B. 36, 0, 32)
      rest: number;             // aktuellerWert - produkt (z.B. 0, 3, 0)
    }
    ```
    Endrest (letzer Rest ≠ 0) wird separat gespeichert.

    **Rendering: CSS-Grid Kästchenpapier**
    - Blaues Raster (border-sky-200, bg-sky-50/30)
    - Jedes Kästchen 32×32px, monospace font
    - Header-Zeile: T H Z E (über Dividend UND über Ergebnis)
    - Zeile 1: Dividend-Ziffern | : | Divisor | = | Ergebnis-Kästchen (Input)
    - Folgezeilen: − Produkt, Rest, runtergeholt, − Produkt, Rest, ...
    - Aktives Kästchen: teal border, auto-focus
    - Gelöste Kästchen: Ziffer angezeigt
    - Zukünftige Kästchen: leer/grau

    **Interaktion:**
    - State: `aktuellerSchritt` (0-basiert), `phase` ('ergebnis' | 'rest')
    - Phase 'ergebnis': Kind tippt Ergebnis-Ziffer → Prüfung → wenn richtig: System zeigt Produkt, Phase wechselt zu 'rest'
    - Phase 'rest': Kind tippt Rest → Prüfung → wenn richtig: System holt nächste Ziffer runter, nächster Schritt
    - Letzter Schritt: wenn Endrest ≠ 0 → Phase 'endrest', Kind tippt Rest
    - Am Ende: onRichtig()

    **Props:**
    ```ts
    interface SchriftlicheDivisionProps {
      dividend: number;
      divisor: number;
      /** Optional: Multiple-Choice für den ersten Schritt (AC-2c) */
      ersteZahlAuswahl?: number[];
      onRichtig: () => void;
      onFalsch: () => void;
    }
    ```

    **Multiple-Choice erster Schritt (AC-2c):**
    Wenn `ersteZahlAuswahl` gesetzt: Zeige vor dem Kästchenpapier Buttons mit Optionen.
    Kind wählt z.B. 45 → System setzt 45 als erste Teilung (4680:5 → erste Stelle: 4→45:5=9).
    Danach: normaler Kästchenpapier-Flow für restliche Stellen.

  <name>Task 2: HalbschriftlicheDivision — Zerlegungs-Eingabe</name>
  <files>src/aufgaben/views/HalbschriftlicheDivision.tsx</files>
  <action>
    Neue Komponente `HalbschriftlicheDivision` erstellen.

    **Layout (Tafel-Stil wie DivisionsZerlegung):**
    ```
    ┌─────────────────────────────────┐
    │  376 : 8 = [___]                │  ← Gesamtergebnis (auto-berechnet)
    │                                 │
    │  [___] : 8 = [___]              │  ← Zeile 1: Kind tippt Teilzahl + Ergebnis
    │   56  : 8 = [___]              │  ← Zeile 2: Rest auto, Kind tippt Ergebnis
    └─────────────────────────────────┘
    ```

    **State:**
    ```ts
    interface Zeile {
      teilDividend: number | null;  // vom Kind eingegeben
      teilErgebnis: number | null;  // vom Kind eingegeben
    }
    ```
    - `zeilen: Zeile[]` — dynamisch wachsend
    - `restDividend: number` — aktuell verbleibender Dividend

    **Flow:**
    1. Zeile 1 aktiv: Zwei Eingabefelder nebeneinander
       - Links: Teilzahl (z.B. 320)
       - Rechts (nach `: 8 =`): Teilergebnis (z.B. 40)
    2. Validierung bei Eingabe des Teilergebnisses:
       - Teilzahl muss ≤ restDividend sein
       - Teilzahl muss Vielfaches des Divisors sein
       - Teilergebnis muss = Teilzahl / Divisor sein
    3. Wenn korrekt: Rest berechnen, nächste Zeile zeigen
    4. Wenn Rest = 0: Fertig → Gesamtergebnis berechnen → onRichtig()
    5. Wenn Rest > 0 und Rest < Divisor: Rest ist Endrest, Aufgabe hat Rest
    6. Wenn Rest ≥ Divisor: nächste Zeile, Rest wird zum neuen Dividend

    **Mehrere Zerlegungen akzeptieren:**
    - 376:8 kann als 320+56 ODER 240+136 ODER 160+160+56 zerlegt werden
    - Validierung prüft nur: Vielfaches + korrekte Teilrechnung
    - NICHT: "muss die Buch-Lösung sein"

    **Props:**
    ```ts
    interface HalbschriftlicheProps {
      dividend: number;
      divisor: number;
      /** Optional: Vielfache zur Auswahl für den ersten Schritt (AC-2b) */
      vielfacheAuswahl?: number[];
      onRichtig: () => void;
      onFalsch: () => void;
    }
    ```

    **Multiple-Choice Vielfache (AC-2b):**
    Wenn `vielfacheAuswahl` gesetzt:
    - Zeige Buttons über der Zerlegung: z.B. `[80] [160] [240] [320] [400] [480]`
    - Kind tippt auf ein Vielfaches → wird als Teilzahl in Zeile 1 eingetragen
    - Danach: normaler Zerlegungs-Flow (Teilergebnis tippen, Rest berechnen)
    - Buttons verschwinden nach Auswahl
  </action>
